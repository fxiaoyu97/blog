# git如何忽略不需要提交的文件或目录

有时候一些有关编辑器的配置文件，打包输出的文件等，不需要提交到远程仓库，这时候我们就需要把文件或者目录添加到忽略文件.gitignore，但是 .gitignore只对没有版本控制的文件起作用。即还没有执行过add操作的文件，只要把名字添加到.gitignore文件即可。另外注意一点，**.gitignore对添加的文件名或者目录名支持正则表达式的匹配。如下所示：

忽略 a.txt 文件
a.txt
忽略a文件夹下a开头的txt文件
/a/a*.txt

3

接下来我们看看下面几种具体情况来分析：

  1. 已经add未commit的文件
  2. 已经commit未push的文件
  3. 已经push的文件

遇到以上情况时，可以考虑先把文件还原到工作区，脱离版本控制。然后再添加到.gitignore文件中。

如何把文件还原到工作区可以参考《如何撤销已经添加、提交、推送的文件或者目录》
